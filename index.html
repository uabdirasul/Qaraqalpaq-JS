Variable

Ózgeriwshiler

Hár qanday dástúr islewi ushın maǵlıwmat kerek.

Máselen hár birimiz kúndelikli paydalanatuǵın "Telegram" dástúri. Telegramdaǵı doslarımız atları, olar jazǵan xabarlar, "profıl" súwretleri bulardıń barlıǵı maǵlıwmat.

Ózgeriwshiler áyne usı maǵlıwmatlardı saqlawǵa járdem beredi.

Ózgeriwshi

Ózgeriwshi bul ózinde maǵlıwmat saqlaytuǵın “qutı”. Ózgeriwshilerge hár qıylı maǵlıwmatlardı saqlay alamız.

Tómendegi bir qatar kod message atlı ózgeriwshi jaratadı.

let message;

Endi bul ózgeriwshiniń ishine = operatorı arqalı maǵlıwmat saqlay alamız:

let message;
message = “Sálem”;

Al endi joqarıdaǵı tekst kompyuter yadına saqlandı hám ol ózgeriwshi menen baylanıstırıldı. Bul tekstten keyinshelik paydalanıw ushın ol menen biriktirilgen ózgeriwshiniń atınan paydalanamız:

let message;
message = “Sálem”;
alert(message) // ózgeriwshiniń ishindegi maǵlıwmattı kórsetedi.

Onı qısqartıw ushın ózgeriwshi jaratılǵan hám oǵan maǵlıwmat kiritilgen qatardı bir-birine qosıwımız múmkin:

let message = “Sálem”;

Jáne de birneshe ózgeriwshilerdi de bir qatarǵa jazıwımız múmkin:

let user = “Ibrayım”, age = 24, message = “Sálem”;

Bul qısqa kóriniwi múmkin, biraq tájiriybeli dástúrshiler onday qılıwdı máslahát etpeydi. Kod jazıwdaǵı tiykarǵı qaǵıydalardan biri - kodımız oqıwǵa ańsat bolıwı kerek bolǵanlıgı ushın hár bir ózgeriwshini bólek-bólek qatarlarǵa jazıw kerek.

Bólek qatarlarǵa jazıw úlkenirek kóriniwi múmkin, biraq ol oqıwǵa ańsat:

let user = “Ibrayım”;
let age = 25;
let mesasage = “Sálem”;

Ayırım dástúrshiler tómendegidey bólek qatarlarǵa jazıw usılınan paydalanadı:

let user = “Ibrayım”,
  age = 25,
  message = “Sálem”;

“comma-first” yaǵnıy birinshi útir usılınanda paydalanatuǵınlar bar:

let user = “Ibrayım”
  , age = 25
  , message = “Sálem”;

Texnik tárepten joqarıdaǵı usıllardıń barlıǵı bir jumıstı atqaradı. Olardan paydalanıw ózlerińizdiń talǵamıńız hám estetikańızǵa baylanıslı.

let’tiń ornına var

Eski skriptlerde, let gilt sózi ornına var degen gilt sózden de paydalanıp ózgeriwshi jaratıw múmkin edi:

var message = “Sálem”;

var gilt sózi menen let gilt sózi derlik birdey. var da ózgeriwshi jaratadı, biraq azǵantayózgesheligi bar.

Olar arasındaǵı parıqtı var ushın arnalǵan bólimde kórip shıǵamız.

Real ómirdegi uqsaslıq

Eger variable’dı ústine túkirarlanbas at jazılǵan qutı dep kóz aldimizǵa keltirsek, bul variable tusinigin arqayın túsinip alamız.

Mısal ushın: message atlı ózgeriwshini ishinde “Sálem!” degen jazıw bar, sırtına bolsa message dep jazılǵan qutı dep alatuǵın bolsaq:

Bul qutınıń ishine qálegen túrdegi maǵlıwmatımızdı qosa alamız.

Jáne de onı qálegenimizshe ózgerte alamız:

let message;
message = "Sálem!”;
message = “Dúnya”; // maǵlıwmat ózgerdi
alert(message);

Qutı ishindegi maǵlıwmat ózgertilse, eski maǵlıwmat óship ketedi:

Hám jáne, eki ózgeriwshi yáki “qutı” jaratıp, birinshisinen ekinshisine maǵliwmattı kóshirip de ótkere alamız:

let hello = “Sálem!”;
let message;
message = hello // hello ózgeriwshisinen maǵlıwmattı kóshirip message ózgeriwshisine ótkeredi // endi eki ózgeriwshiniń de maǵlıwmatı birdey
alert(hello);// Sálem
alert(message); // Sálem

Ekinshi márte qáte shıǵaradı

Ózgeriwshi let arqalı tek ǵana bir márte jazılıwı kerek.

Eki márte jazılsa ol qátelikke sebep boladı:

let message = “Sálem!”
// tákirarlanǵan let qátelik keltirip shıǵardı
let message = “Dúnya”; // SyntaxError: ‘message’ has already been declared

Ózgeriwshilerge at qoyıw

JavaScriptte ózgeriwshilerge at qoyıwda eki sheklew bar:

  1. Qoyılǵan at tek ǵana háripler, sanlar yáki $, _ ǵa uqsas belgilerden ibarat bolıwı kerek
  2. Qoyılǵan at sanlar menen baslanbawı kerek

Qoyıw múmkin bolǵan atlarǵa mısal:

let paydalanıwshıAtı;
let test123;

Eger qoyılǵan atımız bir neshe sózlerden ibarat bolsa camelCase tan paydalanıw máslahát etiledi. camelCase usılı bul - sózler izbe-iz jaylasadı, birinshi sózden tısqarı hámme sóz bas hárip penen baslanadı: judaUzınAt

$ hám _ belgilerinen de paydalanıwımız múmkin. Olar da háriplerge uqsaǵan, heshqanday máni bildirmeytuǵın, ápiwayı belgi. Tómendegi atlardan da paydalanıwǵa boladı:

let $ = 1;
let _ = 2;
alert($+_); // 3

Múmkin emes atlar:

let 1a;
let my-name; // Uncaught SyntaxError: Unexpected token '-’

Úlken menen kishiniń parqı bar

jas hám JAS - ekewi basqa-basqa ózgeriwshi etip qabıllanadı.

Tek ǵana Latınsha emes

Latın álipbesindegi háriplerinen basqa (Kirill, Qıtay logogramları) háripler menen de at qoyıwǵa boladı, biraq usınıs etilmeydi:

let имя = “…”;
let 我 = “…”;

Texnik tárepten bul jerde heshqanday qátelik joq, biraq xalıq aralıq qabıl etilgen konvenciyaǵa kóre ózgeriwshilerge tek ǵana inglis tilinde at beriw kerek.

Bul sózlerden paydalanıwǵa bolmaydı

Ózgeriwshige at qoyıwda paydalanıwǵa bolmaytuǵın sózlerdiń dizimi bar. Bul sózlerden paydalana almaymız, sebebi bul sózler JStiń ózin jaratıwda paydalanılǵan.

Máselen: let, class, return, function.

Tómendegi kod sintaksislik qátelik shıǵaradı:

let let = 5; // Uncaught SyntaxError: let is disallowed as a lexically bound name
let return = 5; // Uncaught SyntaxError: Unexpected token 'return’

use strict siz ózgeriwshi

Ádette ózgeriwshiden paydalanbastan aldın, onı gilt sóz benen jaratıp alıw kerek. Biraq eski versiyalarda onı gilt sózsiz, ápiwayı usılda da jaratıw múmkin edi. Eger eski kodlarǵa maslastırıw ushın use strict'siz jaratsaq, bul usıl ele de isleydi:

// bul use strict’siz usıl
num = 5;
alert(num); // 5

use strict ten paydalansaq, bul tájiriybesizlik penen jazılǵan kod - qátelik keltirip shıǵaradı:

“use strict”;
num = 5; // Uncaught ReferenceError: num is not defined

Konstantalar

Konstanta yaǵnıy maǵlıwmatın ózgertirip bolmaytuǵın ózgeriwshi jaratıw ushın let ornına const paydalanıladı:

const myBirthday = “17.04.2006”;

const arqalı jaratılǵan ózgeriwshiler konstantalar delinedi. Olardıń maǵlıwmatın ózgertiwdiń ilajı joq. Ózgertiwge háreket etilse qátelik keltirip shiǵaradı.

const myBirthday = “17.04.2006”;
myBirthday = “17.10.2006”; // Uncaught TypeError: Assignment to constant variable.

Eger dástúrshi jaratpaqshı bolǵan ózgeriwshi maǵlıwmatın keyinshelik ózgertpewin anıq bilse hám bunı basqa dástúrshilerge de bilmdirmekshi bolsa consttan paydalanıw kerek.

Bas háripli konstantalar

Uzın, yadlap qalıwǵa qıyın nárselerge laqap qoyıp, laqaptı bas hárip penen jazıp, const arqalı konstantalarǵa aylandırıp paydalanıw dástúrshiler ushın jaqsı ámeliyat esaplanadı.

Bunday konstantalarǵa bas hárip hám _ dan paydalanıp at qoyıladı.

Mısal ushın hexademical formatınan paydalanıp reń jaratıp kóreyik:

const COLOR_RED = "#F00";
const COLOR_GREEN = "#0F0";
const COLOR_BLUE = "#00F";
const COLOR_ORANGE = "#FF7F00";
// Usilardan bir ren’di saylaytug’in waqitta
let color = COLOR_ORANGE
alert(color) // #FF7F00

Bul usıldıń paydalı tárepleri:

  1. COLOR_ORANGE #FF7F00 ge qaraǵanda yadlap qalıwg’a ańsatlaw.
  2. di qáte jazıp alıwımız múmkin, biraq COLOR_ORANGE di qáte jazıw imkaniyatı kem.
  3. COLOR_ORANGE #FF7F00 ge qaraǵanda mazmunlıraq. COLOR_ORANGE di oqıǵanda onıń apelsin reń ekenin bile alamız.

Konstantlarǵa qashan tek bas hárip penen at qoyıw kerek hám qashan oǵan normal (qálegen belgi járdeminde) at qoyıw kerek?

Konstanta bul onıń maǵlıwmatı heshqashan ózgermesligin ańlatadı. Biraq, hexademical formatında jaratılǵan, qızıl reńge uqsaǵan - kod orınlanıwınan aldın maǵlıwmatı anıq bolǵan konstantalar yáki kod orınlanıwı waqtında maǵlıwmatı anıq bolatuǵın, biraq keyin ózgermeytuǵın konstantalar da bar.

Máselen:

// pageLoadTime = websayttıń júkleniw tezligi
const pageLoadTime = /* websayttı júklewge ketetuǵın waqıt */

pageLoadTime yaǵnıy websayttıń júkleniwine ketken waqıttı kod orınlanıwınan aldın bile almaymız. Sebebi, onı biliw ushın aldın websayt júkleniwi kerek, júkleniwi ushın kod orınlanıwı kerek. Demek oǵan qádimgidey, ózimiz qálegenimizshe at qoya alamız. Biraq, ol ele konstanta esaplanadı. Sebebi onnan keyin de onıń maǵlıwmatı ózgermeydi.

Basqasha qılıp aytqanda tek ǵana hard-coded yaǵnıy ózgeriwshi ishindegi maǵlıwmattı COLOR_ORANGEdegidey etip ózimiz qoldan jazsaq, oǵan bas háripler menen at qoyıw kerek.

Durıs at qoyiw

Ózgeriwshiler haqqındaǵı jáne bir áhmiyetli nárse bul, durıs at qoyıw.

Ózgeriwshi atı taza hám anıq bir mániske iye, ózi ishinde saqlaytuǵın maǵlıwmatqa túsindirme beretuǵın bolıw shárt.

Ózgeriwshilerge durıs at qoyıw eń áhmiyetli hám eń quramalı uqıp esaplanadı.

Ózgeriwshi atlarına bir kóz taslaw bul kodtıń úyrenshik dástúrshi yáki tájiriybeli dástúrshi tárepinen jazılǵanın ayta aladı.

Real proyektlerde kóp waqıt tazadan kod jazıwǵa emes aldınǵı dástúrshi jazǵan kodtı ózgertiw hám keńeytiwge ketedi. Durıs at qoyıw arqalı kóp waqıttan soń kod jazıwǵa qaytqanda da onı túsiniw ańsat boladı.

Ózgeriwshi jaratıwdan aldın oǵan eń mas keletuǵın at oylap tabıwıńız kerek. Usınday qılıw keyinirek sizge waqıttı tejewge járdem beredi.

At qoyıwda ámel qılınıw kerek bolǵan qaǵıydalar:

  1. Túsiniw ańsat bolǵan atlardan paydalanıń: userName (paydalanıwshı atı), shoppingCart (satıp alınǵan zat sebeti)
  2. Qısqartpa yáki a, b , c ǵa uqsaǵan júdá qısqa atlardan saqlanıń. Bolmasa, ne qılıp atırǵanıńızdı umıtıp qoyasız
  3. Atlardı maksimal xarakteristkalaytuǵın hám qısqa qılıń. Jaqsı emes atlarǵa mısal: data (maǵlıwmat), value (baha). Bunday atlar heshnárseni anıq túsindirmeydi. Bunday atlardı tek ǵana kod konteksti qaysı maǵlıwmat yáki bahaǵa silteme qılıp atırǵanın anıq aytsaǵana qollanıw kerek.
  4. Toparıńız ishindegi hám ózińiz jaratqan shártlerge ámel etiń. Eger saytqa kirgen adam user dep atalǵan bolsa, ózgeriwshi jaratıwda soǵan uqsas qılıp jaratıw kerek. Máselen: currentUser yáki newUser. currentUser hám user, bulardı user sózi biriktirip turıptı. Demek oǵan currentVisitor yáki newManInTown atları durıs kelmeydi.

Juwmaq

Ózgeriwshiler let, var hám const gilt sózleri menen jaratıladı.

  1. let - zamanagóy usıl
  2. var - eski usıl. Ádette onnan paydalanılmaydı. let háım var arasındaǵı ózgesheliklerdi var ushın arnalǵan bólimde kóremiz
  3. const - letke uqsaydı, biraq ol menen jaratılǵan ózgeriwshi maǵlıwmatı ózgermeydi.

Ózgeriwshilerdiń atı ózi ishinde saqlaǵan maǵlıwmattı túsindiretuǵın bolıwı kerek.

Data Types

Maǵlıwmat túrleri

JavaScriptte hárqanday maǵlıwmattıń óz túri boladı. Máselen: string yáki number.

JavaScriptte tiykarǵı 8 maǵlıwmat túri bar. Bul bólimde olardı ulıwmalastırıp kóremiz. Keyingi bólimlerde hár qaysısına bólek toqtalamız.

JavaScriptte ózgeriwshige hár qanday túrdegi maǵlıwmattı saqlay alamız. Máselen: ózgeriwshi birinshi jaratılǵanda string, keyinshelik onı numberge de ózgertiwimiz múmkin:

let message = “Sálem!”;
message = 123456;

Bunday qılıwǵa ruqsat beretuǵın JavaScriptke uqsaǵan tiller “dynamically typed” dep ataladı. “dynamically typed” tilleri ózgeriwshiniń maǵlıwmat túrin qálegenshe ózgertiwge imkan beredi.

Number

let n = 123;
n = 12.345;

Number (san) túri pútin hám bólshek sanlardı óz ishine aladı.

Number (san) túri ushın birqansha operaciyalar bar: * kóbeytiw, / bóliw, + qosıw, - alıw h.t.b.

Ápiwayı sanlardan tısqarı “special numeric values” (arnawlı sanlıq bahalar)ı da bar. Olar: Infinity, -Infinity hám NaN

  1. Infinity matematikadaǵı sheksizlik ∞ ti bildiredi. Bul number túriniń óziniń arnawlı bahası bolıp, ol hárqanday sannan úlken. Bul bahanı hárqanday sandı 0 ge bóliw arqalı alıwımız múmkin: alert(1/0); // Infinity Yáki solayınsha jazıp ta alıwımız múmkin: alert(Infinity); // Infinity
  2. NaN esaplawdaǵı qáteni ańlatadı. NaN nadurıs yáki belgisiz operatsiya nátiyjesinde kelip shıǵadi. (NaN = not a number = san emes) Máselen: alert(NaN + 1); // NaN
    alert(3 * NaN); //NaN
    alert(”not a number” / NaN); // NaN
  3. Matematikalıq operaciyanıń qálegen bir jerinde NaN qollanılsa, juwabı NaN boladı. Biraq tek ǵana NaN ** 0 1 ge teń boladı.

Hár qanday matematikalıq operaciya dástúrdi toqtata almaydı.

JavaScriptte qálegenińizshe matematikalıq operaciyalardı orınlay alasız. Qálegen nárse qıla alasız: nolge bóliw, háriplerdi sanlarǵa qosıw, h.t.b. Hár qanday qáte esap siz shiǵarǵan dástúrdi tap sol qatarda toqtatıp taslay almaydı. Keminde NaN qaytaradı.

"special numeric values" (arnawlı sanlıq bahalar) tek ǵana JavaScripttegi number túrine tiyisli. Onı haqıyqıy matematikada paydalana almaysız.

Sanlar haqqında tolıqlaw Number dep atalǵan bólimde tanısamız.

BigInt

JavaScripttegi number maǵlıwmat túri ózinde 9007199254740991 ((2**53)-1) saınan úlken hám -9007199254740991 sanınan kishi sandı jaqsı saqlay almaydı.

Anıqlaw aytatuǵın bolsaq, JavaScripttegi number maǵlıwmat túri ±((2**53)-1) aralıǵındaǵı sanlardı anıq saqlaydı. Onnan tısqarı sanlar saqlansa precision (sanlardıń xanalar sanı: 12345 - 5 xana) qáteligin keltirip shıǵarıwı múmkin, sebebi onnan keyingi sanlardıń hámmesi de 64-bit bazaǵa mas kele bermeydi.

Mısal ushın tómendegi eki san birdey:

console.log(9007199254740991+1); // 9007199254740992
console.log(9007199254740991+2); // 9007199254740992

Qısqasha aytsaq ((2**53)-1) ǵa deyin bolǵan sanlar number tipinde anıq saqlanadı. Onnan úlken sanlar hárdayım da saqlana bermeydi (ayırım sanlar saqlanadı, ayırım sanlar saqlanbaydı):

console.log(9999999999999999); // 10000000000000000

Joqarıdaǵı kodtıń nátiyjesi log funkciyasınıń ishindegi sandı emes onnan basqa sandı shıǵardı. Bul “saqlay almaydı” degenge mısal boladı.

Bir qansha ápiwayılaw dástúrler ushın ±((2**53)-1) aralıǵındaǵı sanlardıń ózi jetkilikli. Biraq, kriptografiya yáki mikrosekundına shekem anıq saatlar ushın júdá úlken sanlar kerek boladı.

BigInt túri usı sıyaqlı dástúrler ushın kerekli bolǵan júdá úlken sanlardı saqlaw ushın JavaScript tiline qosılǵan.

BigInt túrindegi maǵlıwmattı jaratıw ushın sanlardıń aqırına n qosıladı:

// Sanlardıń aqırındaǵı “n” ol sannıń BigInt ekenin bildiredi
const bigInt = 1234567890123456789012345678901234567890n;

BigInt maǵlıwmat túri dástúrlerde júdá az isletilgeni ushın bul bólimde onı tolıq kórip shıqpaymız. Oǵan bólek bir bólim ajıratıladı.

Maslıq mashqalaları

Házirde BigInt túrin Firefox/Chrome/Edge/Safari browserleri qollap-quwwatlaydı. Biraq IE browseri onı qollamaydı.

Qaysı browserler bul túrdi qollanıwın MDN tolıq kestesinen kórip alıwıńız múmkin.

String

String barlıq belgilerdi (háripler, sanlar, simvollar) óz ishine aladı. Biraq, maǵlıwmatlar tırnaqshalar ishinde jazılıwı kerek.

let str = “Sálem”;
let str2 = “birewlik tırnaqshadan da paydalanıwǵa boladı”;
let phrase = `basqa ${str}di de birge jaylastırıw múmkin`;

JavaScriptte tırnaqshalardıń 3 túri bar:

  1. Ekilik tırnaqsha: “Sálem”.
  2. Birewlik tırnaqsha: ‘Sálem’.
  3. “backtick”lar: `Sálem`.

Ekilik hám birlik tırnaqshalar ápiwayı esaplanadı. Olardıń arasında derlik ózgeshelik joq.

“backtick”lar keńeytirilgen funkcional tırnaqshalar bolıp ol ${…} arqalı teksttiń ishinde ózgeriwshi hám operaciyalardı baylanıstırıwǵa járdem beredi:

let name = “Ibrayım”;
// ózgeriwshi menen baylanıstırıldı
alert(`Sálem, ${name}!`); // Sálem Ibrayım!
// operaciya menen baylanıstırıldı
alert(`juwap ${1+2}ke teń`); // juwap 3ke teń

${…} ishindegi operaciya tekst penen biriktirildi, endi onıń juwabı da teksttiń bir bólegine aylanadı. ${…} diń ishine name sıyaqlı ózgeriwshi yáki 1 + 2ge uqsaǵan matematik operaciyalardı da jaylastıra alamız.

Bunday funkcionallıqtan tek ǵana “backtick”lerde paydalana alamız. Basqa tırnaqshalarda bunday funkcionallıq joq!

alert(”${1+2}tiń juwabı 3 shıǵadı”); // ${1+2}tiń juwabı 3 shıǵadı (ápiwayı tırnaqshalar bunday qıla almaydı)

String haqqında tolıq string ushın arnalǵan bólimde kórip shıǵamız.

Ayırım dástúrlew tillerinde, tek bir qálegen belgi ushın arnalǵan “character” maǵlıwmat túri bar. Máselen: C hám Java dástúrlew tillerinde “char” degen maǵlıwmat túri bar.

JavaScriptte bolsa onday túr joq. Tek ǵana bir túr: string bar. String túriniń maǵlıwmatı bos (” ”) yáki tek bir belgiden (”a”) ibarat bolıwı múmkin.

Boolean (logikalıq túri)

Boolean arqalı tek ǵana eki túrli maǵlıwmattı (bahani) saqlasaq boladı: true hám false.

Bul tip kóbinese awa/yaq sorawlarına juwap beriw ushın paydalanıladı: true - durıs, false - jalǵan degendi bildiredi.

Máselen:

let isMarried = false; // yaq, úylenbegen
let isGraduated = true; // awa, oqıwdı pitkergen

Boolean maǵlıwmatın (bahasın) sanlardı salıstırıp ta alıw múmkin:

let isGreater = 4 > 1;
alert(isGreater); // true (salıstırıw nátiyjesi “durıs” boladı - 4 1den úlken)

Boolean haqqında Logikalıq operatorlar bóliminde tolıq kórip shıǵamız.

“null”

null joqarıdaǵı maǵlıwmat túrleriniń heshqaysısına baylanıslı emes.

Ol óz aldına bólek túr bolıp ol tek null bahasın óz ishine aladı.

let age = null;

JavaScriptte null basqa tillerdegidey ”joq bolǵan object’ke derek” emes.

JSdegi null bos yáki belgisiz maǵlıwmattı bildiredi (maǵlıwmat bar biraq, onıń ishi bos).

Joqarıdaǵı kod age(jas) tıń belgisiz ekenligin bildiredi.

“undefined”

undefinedda óz aldına maǵlıwmat túri.

undefined ózgeriwshige heshqanday maǵlıwmat júklenbegenin, joqlıǵın bildiredi.

Eger ózgeriwshi jaratılıp biraq oǵan heshqanday baha yáki maǵlıwmat berilmese, nátiyje undefined boladı:

let age;
alert(age); // undefined

Texnik tárepten ózgeriwshige maǵlıwmat sıpatında undefined tiń ózin bersek te boladı:

let age = 100;
// maǵlıwmattı undefined qa ózgertemiz
age = undefined;
alert(age); // undefined

Biraq bunday qılıw máslahát etilmeydi. Ádette ózgeriwshiniń ishin bosatıw yáki belgisiz qılıw ushın null paydalanıladı. undefinedti bolsa paydalanıw shárt emes, sebebi standart túrde JStiń ózi heshqanday maǵlıwmat tayınlanbaǵan ózgeriwshilerge usı bahanı biriktirip qoyadı.

Objects and symbols

object basqa túrlerden keskin parıq qıladı.

objectten tısqarı hárqanday túr “primitive” dep ataladı. Sebebi, olar tek ǵana bir zattı saqlaydı (san bolsın, tekst bolsın…). Parıqlaw ushın, objectlerde tek bir emes birqansha maǵlıwmatlardı kirgiziwimiz múmkin.

objectlerge arnawlı bólim ajıratıw kerek. objectlerdi keyingi bólimlerde tolıǵıraq túsinip alamız.

symbol túri objectler ushin birden-bir, tákirarlanbas identifikator jaratıw ushın paydalanıladı. symboldı tolıq túsiniw ushın, birinshi bolıp objectti túsinip alıwımız kerek. Sonıń ushın onı da bólek kórip shıǵamız.

typeof operatorı

typeof operatorı operandtıń maǵlıwmat túrin qaytarıp beredi. Onnan hárqıylı maǵlıwmatlardı qayta islewdi qálegenimizde yáki tek maǵlıwmat túrin biliwdi qálegenimizde paydalanamız:

typeof undefined; // ‘undefined’
typeof 0; // ‘number’
typeof 10n; // ‘bigint’
typeof true; // ‘boolean’
typeof “foo”; // ‘string’
typeof Symbol("id"); // 'symbol'
typeof Math; // 'object' (1)
typeof null; // 'object' (2)
typeof alert; // 'function' (3)

Joqarıdaǵı 3 qatar qosımsha túsindirme talap etedi:

  1. Math bul matematikalıq ámeller ushın arnalǵan “built-in” yaǵnıy JStıń óziniń objecti. Ol haqqında Numbers (sanlar) bóliminde tolıq kórip shıǵamız.
  2. typeof nulldiń juwabı object. Bul typeoftiń rásmiy tán alınǵan qáteligi bolıp ol JavaScripttiń dáslepki kúnlerinde kelip shıqqan. Hám kodlar bir-birine mas keliwi ushın solayınsha qaldırılǵan. Negizinde bul jerdegi typeof null qáte bolıwı kerek!
  3. typeof alerttiń juwabı function boldı. Sebebi alert - bul funkciya (function). Funkciyalar haqqında da arnawlı bólimde tanısamız, hám ol jerde JSta arnawlı function túri joq ekenligin de kóremiz. Funckiyalar (function) object túrine tiyisli esaplanadı. Biraq typeof bizge object emes functiondi qaytardı. Bul qátelik te JStiń dáslepki kúnlerinen kelip shıqqan. Texnik tárepten bunday bolıwı qáte, biraq ámelde qolaylı bolıwı múmkin.

typeof(x) sintaksisi

typeof xtiń basqasha sintaksisinen de paydalanıw múmkin: typeof(x). Bul ekewi de birdey.

Jáne de túsinikli bolıwı ushın: typeof - bul operator, funkciya emes. Bul jerdegi qawsırmalar typeoftiń bir bólegi emes. Bul qawsırmalar matematikalıq gruppalaw ushın paydalanıladı.

Juwmaq

JavaScriptte tiykarǵı 8 maǵlıwmat túri bar.

  • 7 “primitive” túrine kiriwshi tipler:
    • number - hár qıylı sanlar ushın: pútin yáki bólshek sanlar. Pútin sanlar ±((2**53)-1) ge deyin sheklengen
    • BigInt - júdá ulken sanlar ushın
    • string - tekstler ushın. Bul túrdegi maǵlıwmatta heshqanday belgi bolmawı yáki kóp bolıwı múmkin. JSte tek ǵana bir belgi ushın arnalǵan jeke maǵlıwmat túri joq.
    • boolean - true hám false ushın
    • null - belgisiz maǵlıwmatlar ushın - ózinde tek bir bahası bolǵan jeke túr
    • undefined - júklenbegen, tayınlanbaǵan maǵlıwmatlar ushın - ózinde tek bir bahası bolǵan jeke túr
    • symbol - jalǵız, tákirarlanbas identifikatorlar ushın
  • “non-primitive” túrine kiretuǵın jalǵız túr:
    • object - quramalı maǵlıwmat strukturalar ushın

typeof operatorı ózgeriwshi ishindegi maǵlıwmattıń túrin anıqlawǵa járdem beredi.

  • Ádette typeof x kórinisinde jazıladı, biraq typeof(x) qılıp ta paydalanıw múmkin
  • typeof maǵlıwmattıń túrin tırnaqshalar ishinde qaytarıp beredi: “string”.
  • typeofti null menen paydalansaq nátiyjede object shıǵadı. Bul rásmiy túrde tildiń qáteligi esaplanadı. Negizinde ol object emes.
Type Conversions

Maǵlıwmat túrin ózgertiw

Kóbinese, operatorlar hám funkciyalar ózine berilgen maǵlıwmattıń túrin avtomatik túrde durıs hám kerekli bolǵan túrge almastıradı.

Máselen, alert funkciyası nátiyjeni kórsetiw ushın ózine berilgen hárqanday maǵlıwmattı string túrine ótkerip aladı. Matematik ámeller bolsa maǵlıwmatlardı number túrine ózgertedi.

Objectler haqqında keyinirek

Bul bólimde objectlerdi kórip shıqpaymız. Házirshe tek primitive túrine kiretuǵın tiplerdi kórip shıǵamız.

objectlerdiń maǵlıwmat túri o’zgeriwi haqqında "Object to primitive conversion" bóliminde kórip shıǵamız.

String’ge aylandırıw (String Conversion)

Eger maǵlıwmat bizge string formatında kerek bolatuǵın bolsa String Conversion (String’ge aylandırıw) júz beredi.

Mısalı: alert(x) - bul funkciya maǵlıwmattı stringge ózgertedi.

Jáne de, maǵlıwmattı string tipine aylandırıw ushın String(x)de paydalana alamız:

let value = true;
alert(typeof value); // boolean
value = String(value); // endi true stringge aylandi
alert(typeof value); // string

Number’ge aylandırıw (Numeric Conversion)

Matematikalıq funkciya yáki ámellerde Numeric Conversion avtomatik túrde ámelge asadi.

Máselen, / belgisi number túrine kirmeytuǵın maǵlıwmat penen paydalanılǵanda:

alert("6" / "2"); // 3, stringler number túrine avtomatik túrde almastırıldı

Maǵlıwmat túrin number ge ózgertiwde Number(x) ten de paydalanıwımız múmkin:

let str = "123";
alert(typeof str); // string
let num = Number(str); // "123" stringnen numberge aylandı: 123
alert(typeof num) // number

Ádette bunday usıldan (Number(x)) stringge tiykarlanǵan derekten maǵlıwmattı alatuǵınımızdı bilsek, biraq onı number túrinde kiritiliwin qálegenimizde paydalanamız.

Eger string ishindegi maǵlıwmattı number túrine aylandırmaqshı bolsaq biraq, string ishindegi maǵlıwmat san bolmasa nátiyjede NaN juwabın qaytadı:

let age = Number(”san ornına háripler jazıldı”);
alert(age); // NaN, basqa túrge aylandırıw sátsizlikke ushıradı

number túrine aylandırıwdaǵı qaǵıydalar:

Kiritilgen maǵlıwmat Qaytatuǵın maǵlıwmat
undefined NaN
null 0
true hám false 1 hám 0
string Basındaǵı yáki aqırındaǵı boslıqlar (bos jerler, jańa qatarlar, h.t.b) alıp taslanadı. Qalǵan string (tekst)tiń ishi bos bolsa, nátiyje 0 ge teń boladı. Keri jaǵdayda bolsa qátelik bolıp, NaN qaytadı.

Mısallar:

alert(Number(" 123 ")); // 123
alert(Number("123z")); // NaN (z'ti number túrine aylandıra almaydı)
alert(Number(true)); // 1
alert(Number(false)); // 0

Hárdayım yadıńızda tursın: null hám undefinedti number túrine almastırıwǵa háreket etilse, olar hárqıylı nátiyjeler qaytaradı. null 0ge, undefined NaNǵa aylanadı.

Boolean’ge aylandırıw (Boolean Conversion)

Boolean túrine aylandırıw eń ańsatı esaplanadı.

Boolean túrine ózgertiwde logikalıq ámellerden paydalanıladı. Biraq Boolean(x)ten de paydalanıp maǵlıwmat túrin ózgertiwimiz múmkin.

Túr ózgertiwdegi qaǵıydalar:

  • “bos string”, 0, null, undefined hám NaN - false bahasına ózgeredi
  • Odan tısqarı bahalar true ge ózgeredi

Mısal ushın:

alert(Boolean(1)); // true
alert(Boolean(0)); // false
alert(Boolean("sálem")); // true
valert(Boolean("")); // false

string formatındaǵı 0 (”0”) 1ge ózgeredi

PHPge uqsaǵan tillerde “0”, Boolean túrine ózgertilgende false qaytarıladı. Biraq JavaScriptte bos jerden ibarat string bolsa da ol true boladı:

alert(Boolean("0")); // true
alert(Boolean(" ")); // bos jerler de true boladı

Juwmaq

Eń kóp paydalanılatuǵın konvertaciyalar: string’ge, number’ge ha’m boolean’ge aylandırıw.

String Conversion (string túrine ózgertiw) - alert’ge uqsaǵan nátiyjesi string túrinde qaytatuǵın funkciyalarda júz beredi. Oǵan maǵlıwmattı qanday túrde bersek te nátiyjesin kórsetiw ushın onı ózi string'ge avtomatik ózgertip aladı. Yáki onı ózimiz String(x) arqalı da ózgertsek boladı.

Numeric Conversion (number túrine ózgertiw) - matematikalıq ámellerde avtomatik júz beredi. Sonday-aq, ózimiz Number(x) ten paydalanıp ózgertiwimiz mu’mkin.

number túrine ózgertiwdegi qaǵıydalar:

Kiritilgen maǵlıwmat Qaytatuǵın maǵlıwmat
undefined NaN
null 0
true hám false 1 hám 0
string Basındaǵı yáki aqırındaǵı boslıqlar (bos jerler, jańa qatarlar, h.t.b) alıp taslanadı. Qalǵan string (tekst)tiń ishi bos bolsa, nátiyje 0 ge teń boladı. Keri jaǵdayda bolsa qátelik bolıp, NaN qaytadı.

Boolean Conversion (boolean túrine ózgertiw) - logikalıq ámellerde avtomatik júz beredi. Yáki ózimiz Boolean(x) arqalı da ózgerte alamız.

boolean túrine ózgertiwdegi qaǵıydalar:

Kiritilgen maǵlıwmat Qaytatuǵın maǵlıwmat
0, null, undefined, NaN, “0” false
basqa qálegen maǵlıwmat true

Bul qaǵıydalardıń kóbisi túsiniw hám yadlap qalıwǵa ańsat. Biraq kóbinese umıtılıp qalatuǵın qaǵıydalar bar:

  • undefined number túrine ózgertilgende nátiyje 0 emes, NaN boladı.
  • “0” hám “ ” ge uqsaǵan bos orınlı stringler boolean túrine ózgertilgende nátiyje true boladı.

Bul bólimde object konvertaciyası haqqında heshqanday maǵlıwmat berilmegen. Onı objectler haqqındaǵı tiykarǵı nárselerdi bilip alǵannan keyin, kórip shıǵamız.

Basic operators, maths

Tiykarǵı ámeller, matematika

Mektep sabaqlıǵınan kóplegen matematikalıq ámellerdi bilemiz: + qosıw, - alıw, * kóbeytiw, / bóliw.

Bul bólimdi ápiwayı ámeller menen baslap alamız, keyin JavaScripttiń arnawlı táreplerine itibar qaratamız.

“unary”, “binary”, “operand” terminleri

Dawam etiwden aldın, ayırım kóp tákirarlanatuǵın terminler mánisin túsinip alayıq.

  • operand - operatorlar menen qollanıladı. Máselen, 5 * 2 kóbeytmesinde eki operand bar: sheptegi operand - 5, ońdaǵısı 2. Bazı waqıtları bulardı “operand” ornına “argument” dep te ataymız.
  • Eger ámelde tek bir operand bolsa onıń operatorı - unary operator dep ataladı. Máselen, tómendegi - operatorı unary boladı sebebi ámelde tek bir operand bar: let x = 1;
    x = -x;
    alert(x); // -1, unary minus-oń - sandı teris sanǵa aylandırdı
  • Eger ámelde eki operand bolsa onıń operatorı binary boladı: let x = 1, y = 3;
    alert(y - x); // 2, binary minus - sanlardı bir birinen aldı

Matematika

Tómendegi matematik operaciyalardı JavaScriptte paydalana alamız:

  • Qosıw - +,
  • Alıw - -,
  • Kóbeytiw - *,
  • Bóliw - /,
  • Qaldıqtı anıqlaw (remainder) - %,
  • Dárejeni anıqlaw (exponentiation) - **.

Qaldıqtı anıqlaw % (remainder)

% operatorı sırtqı kórinisine qaramay onıń JavaScriptte payızlarǵa baylanıslı jeri joq.

a % bnıń nátiyjesi - anı bǵa bólgendegi qaldıq boladı.

Mısalı:

alert(5 % 2); // 1, 5ti 2ge bólsek qaldıq 1 shıǵadı
alert(8 % 3); // 2, 8di 3ke bólsek qaldıq 2 shıǵadı
alert(8 % 4); // 0, 8di 4ke bólsek qaldıq shıqpaydı

Dárejeni anıqlaw ** (exponentiation)

a ** b nıń nátiyjesi - anıń binshi dárejesi boladı.

Mısalı:

alert(2 ** 2); // 2² = 4
alert(2 ** 3); // 2³ = 8
alert(2 ** 4); // 2⁴ = 16

Bul operatordı tek ǵana pútin emes, bólshek sanlar menen de paydalana alamız:

alert(4 ** (1/2)); // 2
alert(8 ** (1/3)); // 2

Binary + operatorı járdeminde tekstlerdi bir-birine qosıw

Ádette qosıw operatorı + sanlardı bir-birine qosadı.

Biraq binary + string (tekst)ler menen paydalanılsa ol tekstlerdi bir-birine birlestiredi:

let s = "my" + "string";
alert(s); // mystring

Yadıńızda bolsın, eger operandlardıń birewi string bolsa, ekinshi operand ta string’ge ózgertiledi:

alert('1' + 2); // 12
alert(2 + '1'); // 21

Qaysı operand string bolıwınıń áhmiyeti joq, meyli ol birinshi bolsın yáki ekinshi ol báribir string’ge ózgeredi.

Tómende sál quramalıraq mısal berilgen:

alert(2 + 2 + '1'); // 41

Bul jerde operatorlar izbe-izlikte orınlandı. Birinshi operator 2ni 2ge qostı nátiyje 4, keyingi operator bolsa oǵan string túrindegi 1 di qostı, nátiyjede 4 + ‘1’ = 41.

alert('1' + 2 + 2); // bul 14 emes 122 boladı.

Bul jerde birinshi operand string, hám compiler keyingi operandlardı da stringge ótkeredi. 2 ‘1’ge qosıladı, demek nátiyje ‘1’ + 2 = ‘12’ hám ‘12’ + 2 = ‘122’.

Binary + operatorı string ler menen usılayınsha islesetuǵın birden-bir operator. Basqa arifmetik operatorlar tek ǵana sanlar menen isleydi yáki string ishinde tek san bolsa onı number túrine ózgertedi:

alert(6 - '2'); // 4, '2' number túrine ózgertildi
alert('6' / '2'); // 3, eki operand ta number túrine ózgertildi

Number túrine ózgertiw, unary + operatorı

+ eki túrde ushırasadı: binary + hám unary +.

unary + yáki basqasha qılıp aytqanda tek bir operandqa qollanılatuǵın plus operatorı sanlarǵa heshqanday tásir ótkermeydi. Biraq eger operand number túrinde bolmasa, bul operator onıń maǵliwmat túrin number túrine ótkeredi.

Máselen:

// sanlarǵa heshqanday tásir qılmaydı
let x = 1;
alert(+x);// 1
let y = -2;
alert(+y); // -2

// maǵlıwmat túri number emeslerdi number ge aylandıradı
alert(+true); // 1
alert(+""); // 0

Negizinde unary + Number(x) penen teńdey jumıs atqaradı, biraq unary + qısqaraq.

Kóbinese dástúr ishinde string formatındaǵı maǵlıwmattı number túrine ótkeriw kerek bolıp qaladı. Mısalı: HTML form’nan maǵlıwmat alǵan waqıtta, sebebi olar kóbinese string bolıp keledi. Eger sol maǵlıwmatlardı bir-birine qosıw kerek bolsa ne qılamız?

Binary + sol maǵlıwmatlardı string sıpatında bir-birine qosadı:

let apple = "2";
let oranges = "3";
alert(apple + oranges); // 23

Eger biz maǵlıwmattı number sıpatında bir-birine qosıwdı qálesek, birinshi gezekte onıń túrin numberge ózgertiwimiz keyin bir-birine birlestiriwimiz kerek:

let apple = "2";
let oranges = "3";

// ekewiniń de maǵlıwmat túri binary +tan aldın ózgertildi
alert(+apple + +oranges); // 5

// uzınlaw varinantı
// alert(Number(apples) + Number(oranges)); // 5

Matematikalıq tárepten + plustın bunday bolıwı birtúrli bolıwı múmkin. Biraq dástúrshilik tárepten heshqanday túsiniksiz nárse joq: birinshi, unary + stringlerdi numberlerge ózgertedi, keyin binary olardı bir-birine qosadı.

Ne ushın unary +, binary + den aldın orınlanadı? Oǵan sebep - bul olardıń orınlanıw tártibinde (prioritet).

Operatordıń orınlanıw tártibi

Eger ańlatpada birden kóp operator qollanılsa, orınlanıw tártibi olardıń ústinligine yáki basqasha qılıp aytqanda operatorlardıń standart ústin turatuǵınlıǵına baylanıslı boladı.

Mektepten bilemiz 1 + 2 * 2 bul ańlatpada birinshi bolıp kóbeytiw keyin qosıw orınlanıwı kerek. Bul nárse orınlanıw tártibi delinedi. Kóbeytiwdiń orınlanıw tártibi qosıwdikine qaraǵanda joqarıraq.

Qawsırmalarǵa heshqanday orınlanıw tártibi tásir ótkere almaydı. Qawsırmanıń ishindegi ámel birinshi orınlanadı: (1+2) * 2.

JavaScriptte kóplegen operatorlar bar. Hám, hárbir operatordıń óziniń orınlanıw tártip sanı bar. Joqarı sanlılar birinshi orınlanadı. Eger bul sanlar teńdey bolsa, ámeller shepten ońǵa qaray izbe-iz orınlanadı.

Tómende orınlanıw tártip sanı keltirilgen (siz bulardıń hámmesin yadlawıńız shárt emes, biraq unary operatorlar binary operatorlardan aldınlaw orınlanıwın biliwińiz kerek.)

Precedence (orınlanıw tártip sanı) Atı Operatorı
… … …
14 unary qosıw +
14 unary alıw -
13 exponentiation (dárejeni esaplaw operatorı) **
12 kóbeytiw *
12 bóliw /
11 qosıw +
11 alıw -
... ... ...
2 tayınlaw operatorı (teń) =
... ... ...

Kórip turǵanıńızday unary plusdıń orınlanıw tártip sanı 14 bolıp ol binary plus dan úlkenlew. Sonıń ushın +apples + +oranges mısalında unary plus birinshi orınlandı.

Tayinlaw operatori

Sonı da aytıp ótiwimiz kerek = de operator esaplanadı. Bul belgi orınlanıw tártip sanı kestesinde (precedence table) tómen bahaǵa iye yaǵnıy 2.

Sonıń ushın, x = 2 * 2 + 1 sıyaqlı ózgeriwshini jaratqanımızda, birinshi ámeller orınlanadı sońınan ol ózgeriwshige júklenedi.

let x = x = 2 * 2 + 1;

alert(x); // 5

Tayınlaw operatorı = baha (maǵlıwmat) qaytaradı

JavaScripttegi barlıq operatorlar baha qaytaradı. +, - jáne de = de bul qásiyetke iye:

let a = 1;
let b = 2;

let c = 3 - (a = b + 1);

alert(a); // 3
+alert(c); // 0

Joqarıdaǵi kodtaǵı (a = b + 1) ámeliniń nátiyjesi aǵa júkletiledi (3ke teń boladı). Hám onnan keyingi ámellerde de paydalanıladı.

Bunday kodlardıń qalay orınlanıwın bilip alıwımız kerek, sebebi bunday kodlardı JavaScripttiń library’larında ushıratıwımız múmkin.

Jáne de bunday etip kod jazıw máslahát etilmeydi. Bunday jazıw kodtıń oqılıwın qıyınlastıradı.

Shınjırlanǵan tayınlaw

Jáne bir qızıqarlı ózgesheligi - shınjırlanǵan halda maǵlıwmat tayınlaw esaplanadı:

let a, b, c;

a = b = c = 2 + 2;

alert( a ); // 4
alert( b ); // 4
alert( c ); // 4

Shınjırlanǵan tayınlawlar birinshi ońnan shepke qaray orınlanadı. Birinshi, oń táreptegi 2+2 orinlanadı keyin sheptegi c, b hám a ózgeriwshilerine júklenedi. Aqırında, barlıq ózgeriwshiler birdey maǵlıwmat qaytaradı.

Oqılıwı ańsatlaw bolıwı ushın kodtı birneshe qatarlarda jazǵan jaqsıraq:

c = 2 + 2;
b = c;
a = c;

Bul kod oqıw ushın ańsatlaw boldı.

Ornında ózgertiw

Ádette ózgeriwshige operatordi biriktiriw, onnan keyin sol ózgeriwshiniń ózine jańa maǵlıwmattı da júklewimiz kerek.

Mısalı:

let n = 2;
n = n + 5;
n = n * 2;

Bul operatorlardı += hám *= arqalı qısqaraq etip jazıwımız múmkin:

let n = 2;
n += 5; // endi n 7ge teń boldı (n = n + 5 menen birdey nátiyje qaytadı)
n *= 2; // endi n 14ge teń boldı (n = n * 2 menen birdey nátiyje qaytadı)

alert(n); // 14

Bunday qısqa usılda jazıwdı barlıq operatorlar menen paydalanıw múmkin: /=, -= h.t.b.

Bunday etip jazıw - ápiwayı jol arqalı jazıw menen birdey, biraq bul usılda jazılǵan kod eń aqırǵı orınlanadı:

let n = 2;

n *= 3 + 5; // birinshi oń tárep orınlanadı, hám ol n *= 8 benen birdey boladı

alert( n ); // 16

Increment/Decrement (kóbeytiw/azaytıw)

Sandı bir mártege kóbeytiw yáki azaytıw eń keń tarqalǵan sanlı operaciyalardan biri esaplanadı.

Solay etip, bular ushın arnawlı operatorlar da bar:

  • Increment ++ ózgeriwshini bir mártege kóbeytedi: let counter = 2;
    counter++; // counter + 1 benen birdey jumıs atqaradı, biraq qısqaraq
    alert(counter); // 3
  • Decrement -- ózgeriwshini bir mártege kemeytedi: let counter = 2;
    counter--; // counter - 1 benen birdey jumıs atqaradı, biraq qısqaraq
    alert(counter); // 1

Áhmiyetli

Increment hám decrement tek ǵana ózgeriwshilerge qollanıladı. 5++ túrinde qollanılsa qátelik kelip shıǵadı.
